# Security Policy

## Supported Versions
Only the latest version of TKX receives security updates.

## Reporting a Vulnerability
**DO NOT** report security vulnerabilities through public GitHub issues.

Instead, please report them via:
- Email: [Your Email or Security Contact]
- Private security advisory on GitHub

## Responsible Disclosure
We follow responsible disclosure practices:
1. Report vulnerability privately
2. Allow reasonable time for fix
3. Coordinate public disclosure
4. Credit the reporter (if desired)

## Security Features
TKX includes:
- Rate limiting warnings
- False positive filtering
- Legal use warnings
- Ethical guidelines

## Safe Usage Guidelines
1. Always obtain authorization
2. Use on test environments
3. Respect rate limits
4. Follow applicable laws
5. Report vulnerabilities responsibly
